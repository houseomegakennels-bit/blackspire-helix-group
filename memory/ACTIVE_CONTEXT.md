- **Found and fixed a narration text-coverage bug** while producing 11–20: the chapter scene-analysis step's quality gate only checked scene count/average length, so the model could return excerpts that were individually fine but collectively dropped ~20% of each chapter's text (verbatim, in-order fragments with the connective material silently missing) — narration audio is generated straight from that text, so it meant audible content was missing, not just summarized. `analyzeChapters` in `service.ts` now also checks total text coverage and falls back to the full-text mechanical split when the model's scenes don't cover enough of the source. A new `repairSceneNarrationCoverage` function fixed the already-analyzed chapters 11–20 in place (same scene IDs/images, corrected sourceText, stale audio/video cleared for regeneration) — coverage is now 99.7–99.8% across all 20 chapters, and audio/video duration match is exact (0.00s diff) for every chapter.
- **Found and fixed a "chapters won't play in the browser" bug affecting all 20 chapters.** Chapter videos were muxed without `-movflags +faststart`, so the MP4's `moov` atom (the index a browser needs before it can start decoding) landed at the very end of the 30-40MB file instead of the front — a plain progressive `<video src>` tag effectively never loads in that layout. Prior review passes only verified file validity via `ffprobe`, never actual browser video-tag playback, which is why this went unnoticed through two production rounds. Fixed both mux paths in `media.ts` (motion video and the legacy slideshow fallback) for all future renders, added `overwriteAssetBuffer` to `store.ts` plus a `runner-faststart-videos.mts` script that remuxes an existing chapter video in place (`-c copy`, no re-encode, no ID/data changes), and ran it against all 20 already-produced chapters. Verified via both a direct storage read and the live `blackspirehelix.com` endpoint that `moov` now sits right after `ftyp`, and that durations are unchanged.
