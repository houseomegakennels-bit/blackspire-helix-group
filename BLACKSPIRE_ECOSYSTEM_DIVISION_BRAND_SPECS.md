## Purpose

This file translates the `Buyer Engine` visual DNA into sibling specs for:

- `BLACKSPIRE HELIX GROUP`
- `Helix Lawn Command`
- `Blackspire Social OS`
- `Ember Halo`
- `Oracle Helix`

The goal is:

- one ecosystem
- one parent logic
- five distinct business expressions

## Shared Ecosystem Rules

Every Blackspire division should share:

- deep black premium background
- luminous accent energy
- command-center composition
- sharp geometric structure
- signal/intelligence motifs
- strong top-down hierarchy
- elite, system-first tone

Every division should avoid:

- generic startup SaaS styling
- random modern gradients with no structure
- bubbly or playful interface language
- soft corporate blandness
- unrelated visual personalities between products

## Cross-Division Accent Mapping

Use this system consistently:

- `BLACKSPIRE HELIX GROUP` → gold + silver
- `Blackspire Buyer Engine` → orange + gold
- `Helix Lawn Command` → green + gold
- `Blackspire Social OS` → purple + blue
- `Ember Halo` → ember orange + red
- `Oracle Helix` → blue + violet + silver

## Cross-Division Layout Mapping

All division landing pages should use the same skeleton:

1. logo / mark
2. division name
3. tagline
4. short problem statement
5. short solution statement
6. features / capabilities
7. proof / demos
8. CTA

This keeps the ecosystem unified even when the market language changes.

## Cross-Division Component Rules

Shared components should stay structurally the same:

- card shapes
- section rhythm
- nav logic
- panel depth
- border logic
- typography hierarchy

Only these should change per division:

- accent color
- glow color
- icon family emphasis
- symbolic details
- copy language
